Напоминаю последовательность действий для создания репозитория

    1. В PyCharm создать новый проект
    2. Добавить в проект файл .gitignore
        (образец) https://github.com/GennDALF/Python221-project1/blob/master/.gitignore
    3. Опубликовать проект (VCS -> Share on GitHub)
    4. Из репозитория отправить приглашение преподавателю

    (более подробная инструкция в HW_07.10)



  ==========  1  ==========

Добавить в проект файл field.py

В этом файле реализовать функцию show_field(turns) -> None:

    вход: объект с данными о сделанных ходах 
    возврат: None
    stdout: псевдографическое изображение игрового поля со сделанными ходами

Вручную создайте и используйте для тестирования объект с данными о сделанных ходах

Пример того, как может выглядеть игровое поле:
(можно использовать эти или другие символы)

    O |   |   
   ———————————
    O | X |   
   ———————————
    X | O | X



  ==========  2  ========== дополнительная

В файле field.py реализовать функцию check_win(turns) -> bool:

    вход: объект с данными о сделанных ходах 
    возврат: логическое значение, есть ли на поле победная комбинация (столбец/строка/диагональ целиком заполненная одним символом)
    stdout: None



  =========================

Все задания по проекту выполняются в локальном репозитории проекта и загружаются в свой удалённый репозиторий проекта.

В текущий репозиторий помещается TXT файл со ссылкой на репозиторий проекта.

В сервис Журнал отправляется сообщение о выполнении задания со ссылкой на репозиторий проекта.

Функции обязательно документируются.
Параметры функций обязательно аннотируются.

Преподаватель пишет комментарии и правки в коде проекта и обновляет файлы в удалённом репозитории.
Проверяйте обновления в своём репозитории каждый день.
